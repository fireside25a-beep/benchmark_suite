# Deterministic Mathematics Verification Suite

A small, standalone Python experiment that checks fixed mathematical workloads against exact reference results.

The suite performs three deterministic checks:

1. A `2×2` integer matrix multiplication.
2. A `4×4` integer matrix multiplication.
3. A configurable arithmetic workload whose loop result is compared with an independently calculated closed-form result.

The repository includes a shell builder that creates the Python package, configuration file, unit tests, and command-line runner. The builder then compiles the generated Python files, runs the tests, and executes the full suite.

## What this experiment is

This is a compact **deterministic verification suite**. Each workload has a predefined, exact result. A check passes only when the computed result matches its reference value.

It demonstrates:

- reproducible integer calculations;
- exact matrix-product verification;
- comparison of an iterative calculation with a closed-form calculation;
- basic input and matrix-dimension validation;
- automated syntax checks, unit tests, and smoke execution;
- process exit codes that indicate success or failure.

## What this experiment is not

This repository is not:

- an AI or LLM;
- a machine-learning model;
- a general-purpose mathematics library;
- a formal proof system;
- a production performance benchmark;
- a tool that automatically verifies arbitrary external software.

The measured execution times are included only as local observations. They vary by operating system, Python version, processor, system load, and repeated execution. The pass/fail results depend on exact output equality, not on reaching a specific duration.

## Requirements

- Bash
- Python 3.8 or newer

The suite uses only the Python standard library. No `pip install` command or external Python package is required.

## Build and verify the suite

Clone the repository and enter its directory:

```bash
git clone https://github.com/YOUR-USERNAME/YOUR-REPOSITORY.git
cd YOUR-REPOSITORY
```

Make the builder executable:

```bash
chmod +x build_benchmark_suite.sh
```

Run it:

```bash
./build_benchmark_suite.sh
```

The builder will:

1. create the `benchmarks/` package;
2. generate the configuration, modules, runner, and tests;
3. compile every generated Python file with `py_compile`;
4. run four unit tests;
5. execute the complete benchmark suite.

If a `benchmarks/` directory already exists, the builder first copies it to a timestamped backup directory before replacing its generated files.

## Run the suite again

From the repository root:

```bash
python3 -m benchmarks.run_suite
```

On systems where Python 3 is available as `python` instead of `python3`:

```bash
python -m benchmarks.run_suite
```

## Run only the unit tests

```bash
python3 -m unittest discover -s benchmarks/tests -v
```

## Run only the syntax check

```bash
python3 -m py_compile \
  benchmarks/__init__.py \
  benchmarks/modules/__init__.py \
  benchmarks/modules/determinism.py \
  benchmarks/modules/latency.py \
  benchmarks/run_suite.py \
  benchmarks/tests/__init__.py \
  benchmarks/tests/test_suite.py
```

A successful syntax check produces no error output and exits with status code `0`.

## Verification cases

### 1. The 2×2 matrix case

Input matrices:

```text
A = [[1, 2],
     [3, 4]]

B = [[5, 6],
     [7, 8]]
```

Expected product:

```text
[[19, 22],
 [43, 50]]
```

### 2. The 4×4 matrix case

The suite multiplies two fixed `4×4` integer matrices and checks the result against:

```text
[[4, 2, 2, 3],
 [3, 5, 2, 3],
 [6, 4, 6, 5],
 [2, 6, 5, 11]]
```

### 3. The arithmetic workload

For each integer `i` from `1` through `n`, the loop adds:

```text
((3i + 7) × (5i + 11))
```

The explicit loop result is checked against a second implementation based on standard closed-form sums for:

```text
15i² + 68i + 77
```

With the default configuration of `n = 1000`, the exact result is:

```text
5041613500
```

## Configuration

The generated settings are stored in:

```text
benchmarks/config.json
```

Default configuration:

```json
{
  "suite_name": "Deterministic Mathematics Verification",
  "version": "1.0.0",
  "parameters": {
    "stress_iterations": 1000
  }
}
```

`stress_iterations` must be a positive integer. Changing it changes the number of terms evaluated by the arithmetic workload. The iterative and closed-form implementations are both recalculated for the selected value.

## Expected result

A successful default run ends with output equivalent to:

```text
[PASS] 2x2 matrix product
[PASS] 4x4 matrix product
[PASS] algebraic workload
Suite Result: 3/3 checks passed
=== SUITE EXECUTION PASSED ===
```

The displayed durations are expected to differ between runs and computers.

## Failure behavior

The suite exits with:

- status `0` when all checks pass;
- status `1` when a calculated result does not match its expected value;
- status `2` when configuration or input validation fails.

Unit tests also cover rejection of incompatible matrix dimensions.

## Scope and isolation

The generated suite:

- uses no network connection;
- makes no API calls;
- reads only its local `config.json` during normal execution;
- uses no credentials, tokens, datasets, or external services;
- imports only Python standard-library modules and its own generated modules.
